# Лабораторная 3 по СОСам
Запуск: (`cd <директория с .yml файлом>`) (-d для запуска на фоне)
```
docker compose up
```
Остановка: (-v для удаления информации из томов)
```
docker compose down
```

Вопросы:
1) Зачем в Ч.1 использовался --link и почему в Compose он не нужен?
- Потому что контейнеры не знают, как друг друга находить, если не прописать `--link`, а Compose использует .yml файл, где эти связи уже прописаны.
2) Чем отличается image от build в Compose?
- `image` использует готовый скачанный с dockerhub образ, а `build` собирает его из `dockerfile` локально.
3) Почему нельзя масштабировать сервис с публикацией фиксированного порта на одном хосте? Перечислите способы обойти это ограничение.
- Потому что нельзя слушать один и тот же порт нескольким процессам. Обойти это можно указав несколько портов в .yml файле или указать свободный порт т.е. «0».
4) Что делает docker compose down -v и чем отличается от down без -v?
- С `-v` удаляет данные из томом (если те были), а без `-v` сохраняет их содержимое
